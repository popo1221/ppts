代码库

将已有目录初始化成一个Git代码库

git init

新建一个目录，并将其初始化为Git代码库

git init [project-name]

克隆仓库

git clone [url]
git clone [url] dir

配置

显示当前的Git配置

# 显示当前仓库配置
git config --list
# 显示全局配置
git config --list --global

设置配置项

git config --global user.name "gabel"
git config --global user.email "linwei.peng@envisioncn.com"

编辑配置

git config -e [--global]

添加/删除文件

添加指定文件或者目录到暂存区

git add file1
git add file1 file2 file3 ....
git add dir1

添加所有文件到暂存区

git add .

将文件从工作区删除，并且添加到暂存区

git rm file1 file2 dir1

改名文件

git mv file1 file2

代码提交

提交暂存区到仓库区

git commit -m message

提交暂存区的指定文件到仓库区

git commit file1 file2 ... -m message

提交工作区自上次commit之后的变化，直接到仓库区

git commit -a

分支

查看本地分支

git branch

查看远程分支

git branch -r

查看所有分支（包括远程分支）

git br -a

新建一个分支，并切换到新分支

git checkout -b newBranch

新建一个分支，但依然停留在当前分支

git branch newBranch

切换到指定分支

git checkout branch

合并指定分支到当前分支

git merge branch

删除分支

git branch -d branch

除远程分支

git push origin --delete [branch-name]
git branch -dr [remote/branch]

标签（Tag）

查看tag

git tag

基于当前commit创建一个tag

git tag tagname

删除本地Tag

git tag -d tagname

删除远程tag

git push origin :refs/tags/tagname

查看tag信息

git show tagname

提交tag

git push origin tagname

切换到指定Tag

git checkout tagname

查看信息

查看变更

git status

查看变更记录

git log

显示commit历史，以及每次commit发生变更的文件

git log --stat

显示暂存区和工作区的差异

git diff

远程同步

下载远程仓库的所有变动

git fetch branchname

取回远程仓库的变化，并与本地分支合并

git pull [remote] [branch]

上传本地指定分支到远程仓库

git push [remote] [branch]

撤销

恢复暂存区的指定文件到工作区

git checkout file

恢复暂存区的所有文件到工作区

git checkout .

重置暂存区的指定文件，与上一次commit保持一致，但工作区不变

git reset [file]

重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变

git reset --hard [commit]

暂时将未提交的变化移除，稍后再移入

git stash
git stash pop
